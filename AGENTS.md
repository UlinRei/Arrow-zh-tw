# Arrow 專案代理指南

本檔案適用於整個儲存庫。

## 使用者介面語言

- 與使用者溝通時使用台灣繁體中文（zh-TW）。
- 所有授權、提權、風險與等待訊息都必須使用台灣繁體中文。
- 不翻譯命令、參數、路徑、程式碼、設定鍵、工具名稱及權限列舉值；必要時以反引號保留原文。

## 在地化原則

- 所有翻譯必須保持可逆；切換回 `en` 時應顯示原始英文內容。
- 優先使用 `.po`、`TranslationServer` 與 `tr()`，不要直接用中文覆寫英文場景或原始程式字串。
- 使用者建立或修改的專案內容不得因介面語言切換而被覆寫。
- 新專案範例內容只能在建立專案時依目前語系本地化，建立後視為使用者資料。
- `Arrow` 等產品名稱、資源識別碼、節點 ID 與使用者命名內容不應翻譯。
- 節點類型的翻譯位於各 `nodes/<type>/translations/`；主介面翻譯位於 `assets/translations/`。
- 修改 `.po` 後必須讓 Godot 重新匯入，再驗證對應語系。

## 字體

- 語系專用字體不得破壞英文或其他語系的預設字體。
- 繁中目前試用 `assets/fonts/GenSekiGothic2TW-R.otf`。
- 新增字體時應確認散布授權，並在適當的著作權／授權文件中補充出處。
- Windows 匯出設定為 `all_resources`；未使用但位於專案資源中的字體可能增加 `Arrow.pck` 大小。

## 授權文件

- 不得覆寫原始英文 `license` 與 `copyright`。
- 在地化法律內容應使用獨立語系檔，例如 `license.zh_TW`、`copyright.zh_TW`。
- 中文法律內容須標明為參考譯文，若與英文原文不同，以英文原文為準。
- Godot 與第三方授權原文應保留，不應以未經確認的譯文取代。

## Godot 與建置

- 專案使用 Godot 4.7，主要場景為 `res://main.tscn`。
- 修改資源後先執行一次無頭編輯器匯入，再進行匯出。
- Windows Release 輸出位置為 `build/windows/Arrow.exe`。
- Windows 檔案總管圖示使用 `res://icon.svg`；`application/modify_resources` 必須維持為 `true`。
- 建置完成後確認 `Arrow.exe`、`Arrow.pck` 的時間與大小，並檢查 Godot 輸出錯誤。
- 受限執行環境可能無法寫入 `user://logs`；這不等同於一般桌面執行時的權限問題。

## Git 與檔案安全

- 保留使用者既有與未追蹤的變更，不得擅自刪除或納入提交。
- 工作樹混合多項變更時，使用明確檔案路徑暫存，不使用 `git add -A`。
- 翻譯檢查用截圖及未選用字體不得自動提交。
- 提交前執行 `git diff --check`，並以相稱方式完成 Godot 匯入及 Release 匯出驗證。
